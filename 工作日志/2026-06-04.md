#### 一、元灵技能标签系统架构搭建

**需求分析**：
1. 元灵技能可能参与伤害公式计算影响任意变量；或者不参与公式计算，另外效果
2. 系统作用流程：对局中元灵技能被使用 → 系统检测，判断技能的作用对象类型 → 进入类型读取该技能的技能标签 → 对应代码生效效果 → 返回UI对局生效对应效果
3. 按照技能的作用对象类型分为3大类：对球、对场地、对球员

**标签设计**：
- 按对称设计原则：进攻/防守对等（如增伤↔减伤、加速↔减速）
- 保留68个标签（用户确认后重新编号01-68）

| 分类 | 子分类 | 标签数 |
|------|--------|--------|
| BALL | 伤害、速度、范围、轨迹 | 14 |
| FIELD | 障碍、地形、区域、视觉 | 14 |
| PLAYER | 属性、状态、体力、运动、能量、元灵、控制、交互 | 40 |
| **总计** | - | **68** |

**核心组件**：

1. **标签注册表** (`data/spirits/tags_registry.json`) — 68个标签元数据
2. **技能触发器** (`spirit_skill_trigger.gd`) — 检测/冷却/能量/调用标签
3. **标签效果处理器** (`spirit_tag_effect_handler.gd`) — 68个效果函数框架（待实现）
4. **系统管理器** (`spirit_system_manager.gd`) — 统一入口、信号转发

#### 二、快捷设置系统（开发者工具）

**需求**：面向开发者的快捷设置工具，在Godot窗口内运行，用于球员/元灵的增删改。

**功能**：

1. **主入口**：选择"球员管理"或"元灵管理"进入对应子系统

2. **球员管理面板**：
   - 左侧：滚动列表显示所有球员（元素色标）+ 底部"+ 新建球员"按钮
   - 右侧：详情面板（6项属性滑块：体力/防御/速度/攻击/韧性/防御因子）
   - 文本字段：名字/天赋名/天赋描述/大招/描述 + 元素偏好下拉
   - **数值修改** → 解锁滑块 → **确认修改** → 锁定并保存到JSON
   - **新建球员** → 填写表单 → 确认创建 → 写入characters.json

3. **元灵管理面板**：
   - 左侧：滚动列表显示所有元灵 + 底部"+ 新建元灵"按钮
   - 右侧：详情面板（名字/描述/元素/等级/图标色 + 技能列表）
   - **技能新建/编辑弹窗**：
     - 技能名、描述、详细说明、类型（主动/被动）
     - 解锁等级/消耗/能量消耗/冷却时间 滑块
     - **68个标签按钮**按 BALL/FIELD/PLAYER 三类分组展示，点击选中/取消
     - 确认创建后技能写入 skills.json，自动关联到当前元灵

4. **数据同步器** (`dev_data_sync.gd`)：
   - 读写 characters.json / spirits.json / skills.json / tags_registry.json
   - 自动生成新ID（char_XXX / spirit_X / skill_XXX）
   - 创建模板（角色/元灵/技能默认数据）
