## 2026-06-10

### 今日完成

#### 一、球员 Buff 栈系统完整实现（51个 PLAYER 标签）

**目标：** 所有 PLAYER 类标签全部对接，球员属性可通过 buff 修正，支持到期自动恢复

**架构设计：**
- 每个 Player 独立维护 `_buffs` 字典（buff_id → {tag_id, stat, mult, flat, duration, remaining}）
- 属性类 buff：乘法 + 加法修正基础属性，通过 `get_effective_xxx()` 接口读取
- 状态类 buff：布尔标记（眩晕/定身/沉默/缴械/无敌/隐身/免疫控制），到期自动恢复
- 持续效果 buff：每帧 tick 执行回血/掉血
- 目标选择系统：self/enemies/allies/nearest_enemy

**player.gd 新增（+17字段 +31函数）：**

| 类别 | 函数 | 说明 |
|------|------|------|
| Buff 管理 | `add_buff` / `add_status_buff` / `remove_buff` / `clear_all_buffs` | 核心增删 |
| Tick | `_process_buffs` / `_apply_status` | 每帧处理 + 状态切换 |
| 属性接口 | `get_effective_attack/defense/speed/resilience/max_energy` | 基础值 + buff 修正 |
| 状态查询 | `is_stunned/rooted/silenced/disarmed/invincible/stealthed/cc_immune` | 布尔查询 |
| 受伤 | `get_damage_taken_mult` | 无敌=0, 易伤>1 |
| 元灵 | `get/set_skill_cost_mult/cd_mult` + `consume_next_skill_double/half` | 技能修正 |
| 体力 | `set_hp_regen/dot` | 持续回血/掉血 |
| 交互 | `teleport_to` / `return_to_previous` | 传送系统 |

**player.gd 原有代码对接点（4处）：**
1. `_physics_process()` → 新增 `_process_buffs(delta)` + 眩晕/定身移动阻断
2. `take_damage()` → 使用 `get_effective_defense()` + `get_effective_resilience()` + 无敌检查 + 易伤倍率
3. 移动速度 → `get_effective_speed()` 替代裸 `speed`
4. `use_skill()` → 新增沉默检查（被沉默无法使用技能）

**spirit_tag_effect_handler.gd 新增（+51 match + ~15函数）：**

| 标签类别 | 数量 | 分发方式 |
|----------|------|---------|
| 属性(01-16) | 16 | `_apply_player_stat_buff()` 通用函数 |
| 状态(17-20) | 4 | `_apply_player_status()` 通用 / 专用函数 |
| 体力(21-26) | 6 | 独立实现（即时/持续） |
| 运动(27-30) | 4 | stat_buff + 状态 + 专用 |
| 能量(31-38) | 8 | 独立实现 + stat_buff |
| 元灵(39-45) | 7 | 独立实现 |
| 控制(46-49) | 4 | `_apply_player_status()` 通用 |
| 交互(50-51) | 2 | 独立实现 |

**核心通用函数：**
- `_apply_player_stat_buff(params, caster_id, stat, mult, flat)` — 属性 buff 分发
- `_apply_player_status(params, caster_id, status)` — 状态 buff 分发（含免疫控制检查）
- `_get_player_targets(params, caster_id)` — 目标选择

**input_manager.gd 对接：**
- 投球前新增缴械检查（`is_disarmed()` → 阻断投球）

**四链路一致性验证：**
```
tags_registry.json (51 PLAYER tags) → match (51) → handler 函数 → player.gd 接口 (31)
全部一致 ✅
```

**状态到期自动恢复机制：**
```
buff 过期 → remove_buff() → _apply_status(status, false) → 恢复对应布尔标记
特殊：vulnerable_dummy/hp_regen_dummy/hp_dot_dummy/spirit_cost_dummy/spirit_cd_dummy
这些"假状态"过期时自动清除对应的持续效果
```
