#### 三、连按2下自动释放系统

**功能描述：**
- 检测技能键双击（两次按下间隔 < 300ms）
- 双击时自动释放技能，跳过激活阶段
- 单击时激活技能，等待后续操作
- 已激活技能再次单击则取消
- 预留无需鼠标操作的标签判断接口

**SkillStateManager 架构：**

**状态枚举：**
```gdscript
enum SkillState {
    IDLE,       # 空闲
    ACTIVATED,  # 已激活，等待释放
    RELEASING,  # 释放中
    COOLDOWN    # 冷却中
}
```

**核心方法：**
- `setup_player_skills(player_id, skill_ids)` - 设置玩家技能
- `on_skill_key_pressed(player_id, slot)` - 处理按键（检测双击）
- `cancel_active_skill(player_id)` - 取消激活技能
- `get_active_skill(player_id)` - 获取激活的技能
- `is_mouse_required(skill_id)` - 判断是否需要鼠标操作

**双击检测逻辑：**
```gdscript
# 记录每次按下时间
_last_press_times[player_id][slot] = current_time

# 检测时间差
time_diff = current_time - last_press
if time_diff <= 300ms:
    # 双击 → 自动释放
else:
    # 单击 → 激活
```

#### 六、场地物理系统设计讨论

**目标：** 为对场地(FIELD)标签搭建物理底层，确立物理量体系

**确立的物理量：**

| 符号 | 含义 | 默认值 | 单位 |
|------|------|--------|------|
| m | 质量 | 1.0 | 无量纲 |
| v | 速度 | Vector2 | px/s |
| a | 加速度 | F/m | px/s² |
| F | 力 | — | px/s²×m |
| e | 弹性系数 | 0.0 | 无量纲，0~1 |
| μ | 摩擦系数 | 1.0 | 无量纲 |
| g | 重力加速度（标量） | 符号常量 | px/s² |

**设计决策：**

1. **g 无方向** — 2D顶视角，g 仅作为摩擦力公式中的阻力基准值，不产生平面方向力
2. **弹性系数 e 球专用** — 碰撞公式：v'_法线 = -e × v_法线
3. **球员保持速度驱动（方案A）** — 按方向键直接设速度，物理系统（摩擦、冲量、弹性）只在被动情况（被球打中、推力场、碰撞）生效
4. **球员质量用 m 表示** — 参与碰撞和击退计算
5. **技能的"力"由标签预设** — 技能JSON中配置 force 和 duration，用户按键后系统自动应用，不需要用户填值
6. **标签物理量参数化** — 标签的 params 从纯数值扩展为物理量（v_init、a、drag_force、impulse、e 等）

**运动公式：**
```
球员起步：a = F_input / m，v += a × Δt
球员刹车：f = μ × m × g × v̂（反向），v 逐渐减速到 0
球碰撞：v'_法线 = -e × v_法线，v'_切线 = v_切线
```

**g 值建议：~600 px/s²**（球员松手后约0.4秒停下，手感合理）

#### 七、击退物理计算模块

**设计目标：** 解决摩擦系数与击退距离的冲突，实现物理一致性

**核心设计原则：**
1. 击退时间 = 僵直时间（由韧性系统决定，固定）
2. 击退距离响应摩擦系数：`d = d_baseline / μ`
3. 技能标签可提供距离加成（倍率 + 固定值）
4. 韧性系统调用时保证距离准确

**基准参数：**
- 一段击退基准距离：100px（μ=1.0）
- 二段击退基准距离：200px（μ=1.0）
- 标准球速：400px/s

**核心公式：**
```
击退距离 d = (d_baseline / μ) × M_skill + offset
初始速度 v = 2 × d / t_stagger
当前速度 v(t) = v_initial × (1 - t_elapsed / t_total)
```

**主要函数：**
- `calculate_distance()` - 计算击退距离
- `calculate_initial_velocity()` - 计算初始速度（用于动画）
- `calculate_current_velocity()` - 计算匀减速过程中的瞬时速度
- `calculate_knockback()` - 完整计算（一步到位）
- `calculate_required_friction()` - 反向计算：给定目标距离求摩擦系数
- `calculate_required_skill_multiplier()` - 反向计算：给定目标距离求技能倍率
- `generate_distance_samples()` - 生成距离样本表
- `generate_friction_samples()` - 生成摩擦系数影响样本表
- `generate_full_samples()` - 生成完整样本表

**技能参数支持：**
- `skill_distance_multiplier` - 技能距离倍率（如1.5表示加成50%）
- `skill_distance_offset` - 技能固定距离加成（如+50px）
- `ball_speed` - 球速（可选球速加成）
- `enable_ball_speed_bonus` - 是否启用球速加成

**与韧性系统对接：**
```
player.gd:
1. 韧性判定 → 效果类型 + 僵直时间
2. 调用 KnockbackPhysics.calculate_knockback(
       effect_type,
       friction_coefficient,
       stagger_duration,
       skill_multiplier,
       skill_offset
   )
3. 获取 distance 和 initial_velocity
4. 应用击退动画
```

**距离样本（μ=1.0, 僵直0.4s, 无技能加成）：**
| 效果类型 | 击退距离 | 初始速度 |
|---------|---------|---------|
| knockback1 | 100px | 500px/s |
| knockback2 | 200px | 1000px/s |

**摩擦系数影响（knockback1, 僵直0.4s）：**
| 摩擦系数 μ | 击退距离 | 描述 |
|-----------|---------|------|
| 0.5 | 200px | 冰面（翻倍） |
| 1.0 | 100px | 标准地面 |
| 1.5 | 67px | 泥潭（减半） |
| 2.0 | 50px | 粘胶 |

**涉及文件：**
- `scripts/battle/knockback_physics.gd` - 新建

#### 四、标签拆分：百分比/固定值独立

**原则**：每个标签只做一件事，不再用value_type切换

原来"增伤"一个标签内同时支持百分比和固定值 → 拆成"增伤(%)"和"增伤(固定)"两个独立标签

**拆分明细**：
- BALL：增伤/减伤/加速/减速 各拆2个 → +4
- PLAYER属性：攻/防/速/韧各± 各拆2个 → +8
- PLAYER体力：恢复/掉血 各拆2个 → +2
- PLAYER能量：恢复/消耗/最大± 各拆2个 → +4
