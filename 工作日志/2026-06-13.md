### 四、幻象机制

- 视觉：克隆真身头像(数字+背景圆)，颜色改为黯淡(我方蓝→黯蓝alpha0.55/敌方红→黯红)

- 消失：remaining<=0 或 stamina<=0 任一触发，emit illusion_expired → queue_free
- AI对接预留：`is_illusion`/`source_player`/`team`/`is_status_active`(恒false) 字段齐全

## 2026-06-13（bug修复）— 幻象不动 + 清除走鼠标

### Bug 1：幻象不模仿真身运动

**根因**：`illusion.gd` 的 `_physics_process` 只设 `velocity = source_player.velocity`，但**从未调用 `move_and_slide()`**。CharacterBody2D 光设 velocity 不会自己动，必须调 move_and_slide() 才位移（player.gd 在4处都调了）。

**修复**：`illusion.gd` _physics_process 中，镜像/AI移动设完 velocity 后加 `move_and_slide()`。

### Bug 2：清除幻象走了鼠标系统

**根因**：`_clear_illusion_direct` 和 handler `_apply_field_illusion_clear` 都按 clear_all 参数分支，false 时走 `start_clearing`（鼠标选中）。主人要求释放后直接清所有。

**修复**：
- `field_tag_test_panel.gd` `_clear_illusion_direct` → 无条件 `clear_all_illusions()`
- `spirit_tag_effect_handler.gd` `_apply_field_illusion_clear` → 无条件 `clear_all_illusions()`
- `illusion_placer.gd` 的清除模式代码保留（不再被调用，不重构）

### 阶段五：修复组#2 无敌易伤互斥（15/15 全绿）

**根因**（`player.gd` 互斥表 `_MUTEX_LIGHTS` 设计矛盾）：
```gdscript
# 修复前：两条规则互柯打架
"invincible": {"block": ["vulnerable"]},                    # 无敌时易伤被阻（对）
"vulnerable": {"block": [], "clear": ["invincible"]},     # 易伤清除无敌（错！）
```
- `vulnerable` 的 `clear:["invincible"]` 意思是点易伤灯时先清无敌
- 执行顺序：挂无敌→挂易伤→易伤先清无敌再点亮→无敌没了
- 设计意图矛盾：第一条说无敌克制易伤，第二条又说易伤破无敌

**修复**：让易伤走 block，不破无敌
```gdscript
"vulnerable": {"block": ["invincible"]}   # 易伤被无敌阻塞（无敌优先）
```

**验证结果**：重跑组合测试，15/15 全绿
```

═══════ 完成: 15通过 / 0失败 ═══════
```

1. `_load_tag_data`：去掉 FIELD 过滤，加载全部80个，按 BALL/PLAYER/FIELD 三大类分组
2. `_is_tag_implemented`：标注4个未实现标签（terra_change/revert/zone_mark/clear）为false，其余76个可用
3. `_default_target_for_tag`（新增）：PLAYER标签默认目标——增益→self，攻击/debuff→enemies
4. `_get_selectable_tags_for_binding`：返回全部标签

6. `_on_add_binding`：拦截未实现标签

### 阶段八：诊断“增伤打障碍物效果相同”问题（根因：球标签无法重置）

**主人现象**：A增伤(ball_dmg_up_pct)+B建墙，A发球打墙，增伤前后墙剩余血量相同。

**三轮诊断过程**：
1. 模拟消耗：增伤生效（dmg_mult 1.0→1.5，ball_damage 38→57），墙掉 38→50
2. 真实流程诊断：同上，链路正常
3. 完整UI链路诊断（_tag_param_cache + _execute_binding + handler）：完全生效，dmg_mult=1.5，ball_damage=57

**根因**（主人一句“球标签没有技能时间”点醒）：
- 球标签 `_ball_mods` 是**永久修饰符**（无duration），一旦增伤 dmg_mult 永远=1.5
- 测试平台 `_reset_all()`（9键重置）清了球员buff/状态灯，但**漏清了 `_ball_mods`**
- 主人测试对比时：点增伤→发球掉57→按9重置（dmg_mult仍1.5）→再发球还是掉57→两次墙剩相同
- 不是增伤没生效，而是增伤一直挂着无法“关闭”来对比

**修复**（`field_tag_test_panel.gd` 1处）：
- `_reset_all()` 加 `handler.reset_ball_mods()`（清理全部球修饰符）

**标签本身无问题**：增伤逻辑、伤害公式、打障碍物链路全部正常。问题是测试平台的“重置”漏清了球标签。

**球标签 vs 球员标签的生命周期差异**（重要设计认知）：
- 球员标签：有 duration，加进 `_buffs`/`_status_lights`，自动倒计时消失
- 球标签：无 duration，直接改 `_ball_mods`，永久生效直到 `reset_ball_mods()`
