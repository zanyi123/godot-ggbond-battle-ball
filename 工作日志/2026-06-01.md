### 今日完成

#### 一、AI系统6大原则落地修复

**原则4 队形优先——阵型贴墙修复**
- 阵型偏移坐标从负值（贴墙）改为小值（均匀分布半场）
  - offensive: attacker(+60,0), defender(-60,-110), supporter(+20,100)
  - defensive: attacker(+20,60), defender(-80,-80), supporter(-60,80)
  - balanced: attacker(+40,0), defender(-70,-100), supporter(-10,90)
- 半场中心队A=(-190,0), 队B=(190,0)，偏移后 base_pos 落在合理范围

**原则3 个性失误节奏——参数差距大幅拉开**
- attacker: weight_shoot=35, speed_chase_mult=1.1, random_factor=12, hold观察0.15~0.35s
- defender: weight_pass=30, speed_chase_mult=0.9, random_factor=4, hold观察0.5~1.2s
- supporter: weight_pass=40, speed_chase_mult=1.0, random_factor=8, hold观察0.25~0.6s
- 非持球随机因子生效（通过角色分化函数实现不同行为）

**原则2 视野朝向——朝向系统激活**
- 初始朝向按队伍方向（队A朝右，队B朝左）
- register_player 中设置 `facing_direction`
- `_update_facing` 零向量保护
- `_is_in_field_of_view` 零向量保护
- player.gd 中 `_physics_process` 玩家移动时更新朝向
- attacker视野160度/300距离（窄专注），defender视野200度/400距离（宽警惕）

**原则5 对手弱点——行为逻辑落地**
- over_chase: `weakness_overextend=true`，引擎中允许过半场追球
- ball_focused: `weakness_ignore_flank=true`，被侧面近身不躲避
- slow_reaction: think_interval翻2倍，hold_duration翻2倍
- predictable_target: `weakness_stuck_on_target=true`，70%概率重复上次目标

**原则1+6 角色分工——3个队友各自独立**
- 队A默认: [attacker, defender, supporter]，不再是统一supporter
- preparation_ui._rebuild_team_a_profiles 保持各自角色，只改团队策略

#### 二、角色分化行为系统（8个新函数）

| 函数 | 作用 |
|------|------|
| `_decide_off_ball_role` | 无球时按角色选择行为（防御手保护/辅助手接应/主攻手前方等球） |
| `_decide_teammate_has_ball` | 球在队友手里按角色分化跑位 |
| `_get_protect_pos` | 防御手站在持球者身后+偏向敌人横向侧 |
| `_get_assist_pos` | 辅助手站在持球者侧方偏后（横向100px） |
| `_get_attack_wait_pos` | 主攻手前方等球，接近中线时拉横向宽度 |
| `_should_intercept_for_team` | 防御手/辅助手判断是否拦截飞球 |
| `_get_ball_carrier` | 获取己方持球者 |
| `_find_nearest_enemy_to_target` | 找离指定位置最近的敌人 |

#### 三、持球决策增强

- 贴中线且无目标→盲投（不再原地发呆）
- 贴中线且队友太近→盲投
- 看不到敌人且不在边界→前压侦查（dribble_score+40）
- `_eval_shoot` 远距离评分提升，目标待接球状态±15分

#### 四、防卡死循环系统（4层兜底）

| 层 | 机制 |
|----|------|
| 第1层 | `_clamp_forward_to_boundary`: 目标越界→转横向跑位 |
| 第2层 | 5处forward目标改用此函数（观察期/dribble/attack_wait/卡住恢复） |
| 第3层 | 卡住恢复：贴中线→强制投球，非中线→传球或换方向 |
| 第4层 | `_force_redecide_if_at_boundary`: 到达边界目标立即触发下帧重决策 |

#### 五、速度体系统一

**速度计算链（单一数据源）：**
```
characters.json speed (65~85)
  → × SPEED_SCALE (3.25) = player.speed (163~276)
  → 玩家控制用 player.speed
  → AI控制用 player.speed × profile.speed_xxx_mult
  → 冲刺时 +50 px/s
```

- `SPEED_SCALE = 3.25`：最快球员（菲菲85）极速2.75秒跑全场
- AI速度乘数改为 `speed_chase_mult / speed_dribble_mult / speed_move_mult`
- difficulty/weakness 调整也改为调乘数
- register_player 和 update_player_profile 中根据 player.speed × 乘数 计算实际速度

#### 六、冲刺系统（方案B）

- 触发：空手状态左键点击
- 加速量：+50 px/s
- 持续时间：3秒
- 冷却时间：2秒
- 限制：持球时不可冲刺
- 实现：状态标记+常量（不乘speed，无精度漂移）
- player.gd 新增：`is_sprinting / sprint_timer / sprint_cooldown / start_sprint()`
- input_manager.gd：左键空手改为调用 `start_sprint()`，删除旧的 `speed *= 1.5`

#### 七、场地物理重构

**问题1：内外场边界无空气墙**
- 删除内/外场边界所有静态碰撞墙
- 保留场地最外围4面墙（防止飞出场地）
- AI不越线靠逻辑（`_clamp_to_half_field`），不靠物理阻挡
- 球/球员可物理跨越白线（被技能/失误推过线→触发违规判定）

**问题2：外场臂区域连通**
- field_zone.gd 新增 `_draw_outer_field_border`：用 Line2D 画凹字形整体轮廓
- 不再分别画 top_arm/bot_arm/main 的独立边框
- 臂区域与主体之间无白线分隔

**动态隔离墙机制：**
- 开局无隔离墙
- 球员被击败 → `set_penalized(true)` → 碰撞层加layer5 + `_build_penalty_enclosure` 动态创建4面包围墙
- 球员默认 `collision_mask = 1`（不碰隔离墙）
- 被惩罚球员 `collision_mask |= 16`（碰隔离墙）
- 隔离墙只阻挡被惩罚球员，其他球员可自由穿过

**问题3：外场球员可切换控制**
- `_on_player_switch` 无限制（本身就支持切换）
- player.gd `_physics_process` 无 is_penalized 检查（外场球员也能被玩家控制移动）
- 移动范围由 `_clamp_player_position` + 隔离墙保证

**问题4：外场AI与内场同等水平**
- `_decide_penalty_move` 完全重写
- 按角色分化跑位（attacker靠近内场边界，defender偏后，supporter偏侧）
- 防御手/辅助手：用 `_should_intercept_for_team` 拦截飞球
- 移速从 `speed_penalty=120` 改为 `speed_move`（与内场同速）
