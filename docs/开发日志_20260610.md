## 一、球员 Buff 栈系统架构

### 设计原则
- 每个 Player 独立维护 `_buffs` 字典，追踪所有活跃 buff
- 属性类 buff 通过 `mult`（乘法）+ `flat`（加法）修正基础属性
- 状态类 buff 通过布尔标记控制（眩晕/定身/沉默/缴械/无敌/隐身/免疫控制）
- 持续效果 buff（回血/掉血）通过 tick 每帧执行
- 所有有 duration 的 buff 自动到期移除并恢复状态

### Buff 数据结构
```
_buffs = {
    "buff_1": {
        "tag_id": "player_atk_up_pct",    # 来源标签
        "stat": "attack",                  # 修正属性
        "mult": 1.2,                       # 乘法修正
        "flat": 0.0,                       # 加法修正
        "duration": 5.0,                   # 总时长
        "remaining": 3.2,                  # 剩余时长
    },
    "buff_2": {
        "tag_id": "player_stun",
        "stat": "status",
        "status": "stunned",               # 状态标记
        "duration": 2.0,
        "remaining": 1.5,
    }
}
```

| `_is_silenced` | bool | 沉默状态 |
| `_is_disarmed` | bool | 缴械状态 |
| `_is_cc_immune` | bool | 免疫控制状态 |
| `_skill_cost_mult` | float | 技能消耗倍率 |
| `_skill_cd_mult` | float | 技能CD倍率 |

| `_next_skill_double` | bool | 下次技能翻倍 |
| `_next_skill_half` | bool | 下次技能减半 |
| `_hp_regen_rate` / `_hp_dot_rate` | float | 持续回血/掉血速率 |
| `_pre_teleport_pos` | Vector2 | 传送前位置 |

### 新增函数（31个）

**Buff 管理核心：**
- `add_buff()` — 添加属性 buff
- `add_status_buff()` — 添加状态 buff（含免疫控制检查）
- `remove_buff()` — 移除 buff 并恢复状态
- `_process_buffs()` — 每帧 tick（减时/到期/回血/掉血）
- `_apply_status()` — 应用/恢复状态标记
- `clear_all_buffs()` — 清空所有 buff（战斗结束用）

**属性计算接口：**
- `get_effective_attack()` — 基础攻击 + buff 修正
- `get_effective_defense()` — 基础防御 + buff 修正
- `get_effective_speed()` — 基础速度 + buff 修正
- `get_effective_resilience()` — 基础韧性 + buff 修正
- `get_effective_max_energy()` — 基础最大能量 + buff 修正
- `get_damage_taken_mult()` — 受伤倍率（无敌=0, 易伤>1）
- `is_stunned/rooted/silenced/disarmed/invincible/stealthed/cc_immune()` — 状态查询

**元灵系统：**
- `get/set_skill_cost_mult()` — 技能消耗倍率
- `get/set_skill_cd_mult()` — 技能CD倍率
- `add_skill_uses()` — 技能使用次数加成
- `consume_next_skill_double/half()` — 消耗翻倍/减半标记

**体力/能量：**
- `set_hp_regen/dot()` — 持续回血/掉血
- `set_vulnerable()` — 易伤
- `teleport_to()` / `return_to_previous()` — 传送/返回

### 原有代码对接点（4处）
1. `_physics_process()` — 新增 `_process_buffs(delta)` tick + 控制/眩晕移动阻断
2. `take_damage()` — 使用 `get_effective_defense()` + `get_effective_resilience()` + 无敌/易伤检查
3. 移动速度 — `get_effective_speed()` 替代 `speed`

## 三、spirit_tag_effect_handler.gd 新增内容

### 新增 match 分支（51个 PLAYER + 10个 FIELD）

PLAYER 标签按类别分发：

| 类别 | 标签数 | 分发方式 |
|------|--------|---------|
| 属性(01-16) | 16 | `_apply_player_stat_buff()` 通用函数 |
| 状态(17-20) | 4 | `_apply_player_status()` 通用 / 专用函数 |
| 体力(21-26) | 6 | 独立实现（即时/持续） |
| 运动(27-30) | 4 | stat_buff + 状态 + 专用 |
| 能量(31-38) | 8 | 独立实现 + stat_buff |
| 元灵(39-45) | 7 | 独立实现 |
| 控制(46-49) | 4 | `_apply_player_status()` 通用 |
| 交互(50-51) | 2 | 独立实现 |

### 新增核心函数

- `_apply_player_stat_buff()` — 通用属性 buff 分发（attack/defense/speed/resilience/max_energy）
- `_apply_player_status()` — 通用状态 buff 分发（stunned/rooted/silenced/disarmed/invincible/stealthed/cc_immune）
- `_get_player_targets()` — 目标选择（self/enemies/allies/nearest_enemy）
- `_apply_player_vulnerable/reveal/unroot` — 专用状态函数
- `_apply_player_hp_*` — 6个体力函数（即时/持续/百分比/固定）
- `_apply_player_energy_*` — 4个能量函数
- `_apply_player_spirit_*` — 7个元灵系统函数
- `_apply_player_teleport/return` — 传送/返回

### 目标选择设计
```
params.target = "self"          → 施法者自己（默认）
params.target = "enemies"       → 敌方全体
params.target = "allies"        → 己方全体
params.target = "nearest_enemy" → 最近敌方（无则全体）
```
