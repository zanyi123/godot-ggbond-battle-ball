# 第3步：闹钟纸条（持续效果系统）设计

## 概念

### 闹钟纸条 = 每帧执行的定时任务

第1步的buff堆栈改属性值（挂上就生效，到期自动消失）。
第2步的状态灯控制行为（开灯=限制行动，关灯=恢复）。
**第3步不一样**——需要在持续时间内**每帧执行一个动作**。

比如"每秒恢复5体力，持续5秒"：
- 不是一次性+25
- 而是每帧 `stamina += 5 × delta`，跑5秒后自动停

这就是闹钟纸条：**纸条上写着"每帧干什么"，闹钟响了就撕掉纸条**。

---

## 涉及的标签（2个）

| # | 标签ID | 名称 | 每帧干什么 | 默认参数 |
|---|--------|------|-----------|---------|
| 25 | `player_hp_regen` | 持续恢复 | `stamina += rate × delta` | value=5, duration=5s |
| 26 | `player_hp_dot` | 持续掉血 | `stamina -= rate × delta`，到0触发击败 | value=5, duration=5s |

### 为什么只有2个标签需要闹钟纸条？

其他体力标签（21-24）都是**一次性**的：
- `player_hp_heal_pct`：瞬间恢复X%，不需要每帧
- `player_hp_damage_pct`：瞬间扣X%，不需要每帧
- `player_hp_heal_flat`：瞬间+固定值，不需要每帧
- `player_hp_damage_flat`：瞬间-固定值，不需要每帧

只有 **持续恢复** 和 **持续掉血** 需要每帧执行。

---

## 计算公式

### 持续恢复

```
每帧恢复量 = rate × delta
stamina = min(max_stamina, stamina + rate × delta)
```

举例：rate=5, duration=5s
- 每帧（假设60fps，delta≈0.0167）：5 × 0.0167 = 0.083
- 5秒总计：5 × 5 = **25体力**
- 不会超过 max_stamina

### 持续掉血

```
每帧伤害量 = rate × delta
stamina = max(0, stamina - rate × delta)
if stamina <= 0 → 触发击败
```

举例：rate=8, duration=3s，当前stamina=20
- 3秒总计掉血：8 × 3 = **24体力**
- 体力到0时立即触发 `_on_defeated()`

## 闹钟纸条的数据结构

```gdscript
# Player 身上挂着的闹钟纸条列表
var _tick_effects: Dictionary = {}
# 格式: {
#   "hp_regen": { "type": "regen", "rate": 5.0, "remaining": 5.0 },
#   "hp_dot_1": { "type": "dot", "rate": 8.0, "remaining": 3.0 },
# }
```

每张纸条：
| 字段 | 含义 |
|------|------|
| `type` | `"regen"` 或 `"dot"`，决定每帧是加还是减 |
| `rate` | 每秒的量（正数） |
| `remaining` | 剩余秒数，每帧减 delta，到0撕掉 |

## 和前两步的对比

| | 第1步 Buff堆栈 | 第2步 状态灯 | 第3步 闹钟纸条 |
|---|---|---|---|
| 服务标签 | 20个属性 | 11个状态/控制 | 2个持续效果 |
| 执行时机 | 被动（别人问属性值时才算） | 被动（保安检查时才看） | **主动（每帧执行）** |
| 计算 | 基础×连乘+求和 | 只有开/关 | rate × delta |
| 叠加规则 | 可叠加 | 不可叠加（刷新时间） | **可叠加**（两个持续回血共存，各回各的） |

---

## 为什么要可叠加？

同一球员可能同时被两个持续效果影响：
- 己方技能给"持续恢复5/s"
- 敌方技能给"持续掉血8/s"
- 两者共存，每帧净效果 = -3/s

所以用 **Dictionary（id→纸条）** 而不是单开关，允许不同来源的持续效果同时存在。

同id重复添加 → 覆盖旧值（和buff堆栈一致）。

---

## 每帧执行位置

在 `_physics_process` 中，**所有分支都必须tick**（和状态灯一样）：

```
_physics_process(delta):
    冲刺计时更新
    if 击退中:
        tick_闹钟纸条(delta)   ← 加在这里
        return
    if 僵直/眩晕/定身:
        tick_闹钟纸条(delta)   ← 加在这里
        return
    正常移动
    tick_闹钟纸条(delta)       ← 加在这里
```

---

## tick 闹钟纸条的函数

```gdscript
func _tick_effects(delta: float) -> void:
    var to_remove: PackedStringArray = []
    for id in _tick_effects:
        var effect: Dictionary = _tick_effects[id]
        var etype: String = effect.get("type", "")
        var rate: float = effect.get("rate", 0.0)
        
        # 每帧执行
        if etype == "regen":
            stamina = minf(max_stamina, stamina + rate * delta)
        elif etype == "dot":
            stamina = maxf(0.0, stamina - rate * delta)
            if stamina <= 0.0 and not is_defeated:
                _on_defeated()
        
        # 倒计时
        effect["remaining"] = effect.get("remaining", 0.0) - delta
        if effect.get("remaining", 0.0) <= 0.0:
            to_remove.append(id)
    
    for id in to_remove:
        _tick_effects.erase(id)
```

## 对外接口

```gdscript
# 添加一个持续效果
func add_tick_effect(id: String, type: String, rate: float, duration: float) -> void:
    _tick_effects[id] = { "type": type, "rate": rate, "remaining": duration }

# 手动移除
func remove_tick_effect(id: String) -> bool:
    return _tick_effects.erase(id)

# 查询是否存在
func has_tick_effect(id: String) -> bool:
    return _tick_effects.has(id)

# 查询某个类型的总速率（如所有regen加起来每秒回多少）
func get_total_tick_rate(type: String) -> float:
    var total: float = 0.0
    for id in _tick_effects:
        if _tick_effects[id].get("type", "") == type:
            total += _tick_effects[id].get("rate", 0.0)
    return total
```

## 与状态灯的交互

- **无敌灯亮时**：持续掉血（dot）不生效，持续恢复（regen）照常
- **眩晕/定身**：不影响持续效果（持续回血/掉血是被动效果，不需要行动）
- **沉默/缴械**：不影响持续效果

实现方式：在 `_tick_effects` 中加无敌检查：

```gdscript
if etype == "dot":
    if is_status_active("invincible"):
        pass  # 无敌中，不掉血（但倒计时照跑）
    else:
        stamina = maxf(0.0, stamina - rate * delta)
        ...
```

## 测试用例清单

| # | 测试名 | 验证什么 |
|---|--------|---------|
| 1 | 持续恢复 | add(regen, 5/s) → tick(1s) → stamina+5 |
| 2 | 持续掉血 | add(dot, 8/s) → tick(1s) → stamina-8 |
| 3 | 到期停止 | add(regen, 5/s, 2s) → tick(2s) → 纸条撕掉，stamina不再涨 |
| 4 | 叠加共存 | regen(5/s) + dot(8/s) → tick(1s) → 净-3 |
| 5 | 上限保护 | stamina=98, regen(10/s) → tick(1s) → stamina=100（不超过max） |
| 6 | 下限击败 | stamina=5, dot(10/s) → tick(1s) → stamina=0, 触发击败 |
| 7 | 覆盖同id | add(regen, 5/s) → add(同id, regen, 10/s) → 只有10/s |
| 8 | 手动移除 | add → remove_tick_effect → tick不再执行 |
| 9 | 无敌挡dot | 无敌灯亮 + dot(10/s) → tick(1s) → stamina不变（倒计时照跑） |
| 10 | 到期后无敌 | dot(10/s, 2s) + 无敌 → tick(2s) → dot到期消失，无敌灯还在 |

## 待确认

1. **无敌是否挡持续掉血？**
   - 建议：**挡**。无敌=免疫一切伤害，持续掉血也是伤害。

2. **体力到0时是立即击败还是tick结束后击败？**
   - 建议：**立即击败**（帧内检测到≤0就触发）。

3. **多个dot同时把体力打到0，只触发一次击败？**
   - 建议：**是的**，`is_defeated` 检查防止重复触发。
