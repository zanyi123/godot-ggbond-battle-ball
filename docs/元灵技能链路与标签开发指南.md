# 元灵技能链路与标签开发指南

> 生成时间: 2026-06-09
> 用途：快速查阅技能生效链路，高效开发新标签

---

## 一、涉及技能生效的核心系统（6个）

| 系统 | 文件路径 | 职责 |
|------|----------|------|
| **SkillStateManager** | `systems/spirit_system/skill_state_manager.gd` | 按键状态管理（激活/双击释放/取消） |
| **SpiritSystemManager** | `systems/spirit_system/spirit_system_manager.gd` | 技能系统总入口，协调触发器+效果处理器 |
| **SpiritSkillTrigger** | `systems/spirit_system/spirit_skill_trigger.gd` | 技能触发：查CD/能量/标签注册表，分发到处理器 |
| **SpiritTagEffectHandler** | `systems/spirit_system/spirit_tag_effect_handler.gd` | 标签效果执行：所有标签的 match 分支 + 对应函数 |
| **Ball** | `battle/ball.gd` | 技能载体：读取 `_ball_mods` 修饰球属性 |
| **ObstacleManager / FieldPhysicsManager** | `battle/obstacle_manager.gd` / `battle/field_physics_manager.gd` | 场地类标签落地执行者 |

## 二、技能生效完整调用链

### 链路 1：按键激活 → 修饰球属性

```
玩家按 4/5/6 键
  │
  ▼
InputManager._handle_skill_key_press(slot)
  │
  ▼
SkillStateManager
  ├─ 首按 → state = ACTIVATED
  │   └─ emit skill_activated(skill_id, player_id)
  │       │
  │       ▼
  │       Ball._show_skill_aura(skill_data)    ← 球光环显示（元素颜色脉冲）
  │       SpiritTagEffectHandler._ball_mods     ← 写入修饰符
  │
  ├─ 双击(300ms内) → state = RELEASING → 自动发球
  │
  └─ C键取消 → state = IDLE
      └─ emit skill_cancelled
          └─ Ball._clear_skill_aura() + _ball_mods 重置
```

### 链路 2：发球时 → 修饰符写入球属性

```
玩家松开左键（或双击自动释放）
  │
  ▼
InputManager.emit throw_requested(direction, power)
  │
  ▼
BattleManager → Ball.launch(direction, power, attacker)
  │
  ▼
Ball.launch() 内部：
  ├─ 读取 SpiritTagEffectHandler._ball_mods
  │   ├─ ball_damage = attacker.attack_power × dmg_mult + dmg_flat
  │   ├─ ball_speed = base_speed × speed_mult + speed_flat
  │   ├─ max_flight_distance = base × range_mult + range_flat
  │   ├─ trajectory_type = "tracking" / "boomerang" / "straight"
  │   └─ 其他标记（穿透/锁定/AOE）
  ├─ 重置 _hit_player_ids（防重复命中）
  └─ 球开始飞行
```

### 链路 3：技能正式触发（走 SpiritSystem 全链路）

```
Player.use_skill() → emit skill_used(player_id, skill_id)
  │
  ▼
BattleManager._on_player_skill_used()
  │
  ▼
SpiritSystemManager.use_skill(player_id, skill_id)
  │
  ▼
SpiritSkillTrigger.trigger_skill()
  │
  ├─ 检查1：技能是否在 _player_skills 中？（是否装备）
  ├─ 检查2：CD是否就绪？
  ├─ 检查3：能量是否足够？
  │
  ├─ 通过 → 从 DataManager.skills 找到技能数据
  │   │
  │   ▼ 遍历技能的 tags[] 列表
  │   │
  │   ▼ SpiritTagEffectHandler.apply_tag_effect(tag_id, params)
  │     │
  │     ├─ ball_* 标签（01-17）→ 修改 _ball_mods
  │     ├─ field_obs_add       → ObstacleManager.start_placing()
  │     ├─ field_obs_clear     → ObstacleManager 清除
  │     ├─ field_phys_*        → FieldPhysicsManager 修改摩擦/弹性
  │     └─ player_* 标签       → 直接修改 Player 属性
  │
  └─ 未通过 → 打印拒绝原因
```

### 链路 4：球飞行中 → 技能效果实际生效

```
Ball._physics_process(delta)
  │
  ├─ 追踪(tracking)  → _ball_mods.tracking_target 非空 → 每帧转向
  ├─ 回旋(boomerang) → _ball_mods.boomerang → 飞到距离后反向
  ├─ 穿透(penetrate) → _ball_mods.penetrate → 命中后不停
  ├─ 障碍物卡住      → stuck_on_obstacle → 逐帧消耗
  │
  └─ 碰撞球员 → _on_body_entered(player)
      ├─ AOE → _ball_mods.aoe_radius > 0 → 范围伤害
      └─ 命中 → Player.take_damage()
          └─ 韧性判定 → KnockbackPhysics ← FieldPhysicsManager.μ
```

## 三、元灵技能完整生命周期

```
1. 数据层    DataManager 加载 spirits.json / skills.json / tags_registry.json
     ↓
2. 装备层    PreparationUI → Player.equip_spirit() → SpiritSystemManager.set_player_skills()
     ↓
3. 激活层    InputManager → SkillStateManager → _ball_mods 写入 → 球光环显示
     ↓
4. 触发层    Ball.launch() 读取 _ball_mods
             SpiritSystemManager → SkillTrigger(CD/能量) → TagEffectHandler.apply_tag_effect()
     ↓
5. 生效层    Ball 飞行(追踪/回旋/穿透/卡障碍) → 命中(伤害/AOE/韧性/击退)
     ↓
6. 清除层    Ball._on_ball_stopped() → _ball_mods 重置 → 光环消失
```

## 五、新增标签开发检查清单

新增一个标签时，需要改动 **4个位置**，按顺序：

### ✅ Step 1：注册表 — `data/spirits/tags_registry.json`

在 `tags` 数组中添加条目：

```json
{
  "id": "标签ID",
  "name": "显示名",
  "category": "ball / field / player",
  "code": "编号",
  "description": "说明",
  "params": [
    {"key": "参数名", "type": "float", "default": 0, "description": "参数说明"}
  ]
}
```

### ✅ Step 2：效果处理器 — `scripts/systems/spirit_system/spirit_tag_effect_handler.gd`

#### 2a. `_ball_mods` 字典（仅球类标签需要）
在 `_ball_mods` 字典中添加新键 + 默认值

#### 2b. `apply_tag_effect()` match 分支
添加新分支：

```gdscript
"标签ID":
    _apply_标签函数名(params)
```

#### 2c. 实现标签函数
```gdscript
func _apply_标签函数名(params: Dictionary) -> void:
    # 读取参数
    # 修改目标系统
    # 打印日志
    effect_applied.emit("标签ID", params)
```

### ✅ Step 3：载体代码（球/障碍物/球员）

根据标签类别，在目标系统中添加读取逻辑：

- **球类** → `ball.gd` 的 `launch()` 或 `_physics_process()` 中读取 `_ball_mods` 新键
- **场地类** → `field_physics_manager.gd` 或 `obstacle_manager.gd` 中添加新方法
- **球员类** → `player.gd` 中添加属性字段 + 响应逻辑

### ✅ Step 4：四链路一致性验证

```
tags_registry.json 的 id
  ║  一致
SpiritTagEffectHandler apply_tag_effect() match 分支
  ║  一致
SpiritTagEffectHandler _apply_xxx() 函数名
  ║  一致
目标系统读取的参数名
```

## 七、_ball_mods 修饰符完整字段参考

```gdscript
var _ball_mods: Dictionary = {
    # ---- 数值类 ----
    "dmg_mult": 1.0,           # 伤害倍率（pct标签修改）
    "dmg_flat": 0.0,           # 伤害固定加减（flat标签修改）
    "speed_mult": 1.0,         # 球速倍率
    "speed_flat": 0.0,         # 球速固定加减
    "range_mult": 1.0,         # 飞行距离倍率
    "range_flat": 0.0,         # 飞行距离固定加减
    "armor": 0.0,              # 护甲值（抵消伤害）

    # ---- 飞行行为类 ----
    "penetrate": false,        # 穿透（命中不停）
    "tracking_target": null,   # 追踪目标节点
    "tracking_turn_speed": 0.0,# 追踪转向速度
    "boomerang": false,        # 回旋球
    "boomerang_triggered": false,
    "boomerang_return_dir": Vector2.ZERO,
    "boomerang_dist": 0.0,
    "lock_straight": false,    # 强制直线（禁用其他轨迹）
    "spread_done": false,      # 扩散已触发
    "lockon_target": null,     # 精准锁定目标

    # ---- 范围类 ----
    "aoe_radius": 0.0,         # AOE半径
    "aoe_damage_pct": 0.5,     # AOE伤害百分比
}
```
