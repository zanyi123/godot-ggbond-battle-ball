# 第2步：控制状态系统设计（灯+保安）

## 概念

### 灯 = 状态开关

每个球员身上有几盏灯，灯亮=该状态生效，灯灭=正常。

| 灯 | 对应标签 | 亮了会怎样 |
|----|---------|-----------|
| 🔴 眩晕灯 | `player_stun` | 不能移动、不能投球、不能技能 |
| 🔇 沉默灯 | `player_silence` | 不能使用技能 |
| 🚫 缴械灯 | `player_disarm` | 不能投球 |
| 🔒 定身灯 | `player_root` | 不能移动（可以投球和技能） |
| 🛡️ 无敌灯 | `player_invincible` | 受到伤害=0 |
| 👻 隐身灯 | `player_stealth` | 不被敌方锁定 |
| 💥 易伤灯 | `player_vulnerable` | 受到伤害×倍率 |
| 🛡️ 免控灯 | `player_cc_immune` | 免疫眩晕/沉默/缴械/定身 |
| 👁️ 显形灯 | `player_reveal` | 强制敌方隐身者现身（一次性） |

### 保安 = 检查站

球员想做某件事时，先过保安检查对应的灯。

```
球员按技能键 → 保安看沉默灯 → 亮着？→ 拦住，不放行
球员按投球键 → 保安看缴械灯 → 亮着？→ 拦住，不放行
球员按移动   → 保安看眩晕灯和定身灯 → 任何一个亮？→ 拦住
敌方球飞来   → 保安看无敌灯 → 亮着？→ 伤害变0
```

## 灯的数据结构

不用纸条了，灯就是开关+计时器：

```gdscript
# Player 身上的状态灯字典
var _status_lights: Dictionary = {}
# 格式: { "stunned": { "on": true, "remaining": 2.0 }, "silenced": { ... }, ... }
```

为什么不用 buff 堆栈？
- 属性buff可以叠加（多个+30%攻击共存），灯**不能叠加**（眩晕就是眩晕，不会"更眩晕"）
- 灯只有开/关+倒计时，没有乘法加法计算

## 保安检查点（代码插入位置）

### 1. 移动前检查 — `_physics_process`

```
现在：僵直中 → 不能移动
改成：僵直中 OR 眩晕灯亮 OR 定身灯亮 → 不能移动
```

位置：`player.gd` `_physics_process` 里的 `_stagger_timer > 0.0` 判断后面

```gdscript
# 现在
if _stagger_timer > 0.0:
    ... # 僵直中不能动

# 改成
if _stagger_timer > 0.0 or is_status_active("stunned") or is_status_active("rooted"):
    ... # 僵直/眩晕/定身都不能动
```

### 2. 投球前检查 — `input_manager.gd`

```
现在：左键释放 → 直接发 throw_requested
改成：左键释放 → 保安检查缴械灯 → 亮着？→ 拦住，不emit
```

位置：`input_manager.gd` `_on_left_click_release` 里 `throw_requested.emit` 前

```gdscript
# 在 throw_requested.emit 之前加
if controlled_player.is_status_active("disarmed"):
    return  # 缴械中，不能投球
```

### 3. 技能前检查 — `player.gd use_skill`

```
现在：use_skill → 检查CD → 检查能量 → 放行
改成：use_skill → 检查沉默灯 → 检查CD → 检查能量 → 放行
```

位置：`player.gd` `use_skill` 函数开头

```gdscript
# 在 slot_index 检查之后加
if is_status_active("silenced"):
    return  # 沉默中，不能技能
```

### 4. 受伤前检查 — `player.gd take_damage`

```
现在：take_damage → 计算防御 → 计算韧性 → 扣血
改成：take_damage → 检查无敌灯 → 亮着？→ 伤害变0，直接return
```

位置：`player.gd` `take_damage` 函数开头（`is_defeated` 检查之后）

```gdscript
# is_defeated 检查之后加
if is_status_active("invincible"):
    return {"damage": 0, "effect": "none"}  # 无敌，不受伤
```

### 5. 易伤倍率检查 — `player.gd take_damage`

```
现在：直接用 amount 算伤害
改成：amount × 易伤倍率（如果易伤灯亮）
```

位置：`take_damage` 里计算 `effective_amount` 的地方

```gdscript
var dmg_mult: float = 1.0
if is_status_active("vulnerable"):
    dmg_mult = _status_lights["vulnerable"].get("multiplier", 1.5)
var effective_amount: float = amount * dmg_mult
```

## 免控灯的特殊规则

免控灯亮着时，4种控制灯**点不上去**：

```gdscript
func turn_on_light(status_name: String, duration: float, extra: Dictionary = {}) -> bool:
    # 控制类状态，先检查免控
    if status_name in ["stunned", "silenced", "disarmed", "rooted"]:
        if is_status_active("cc_immune"):
            print("[Status] 免控中，%s 无效" % status_name)
            return false  # 点灯失败
    _status_lights[status_name] = { "on": true, "remaining": duration, ...extra }
    return true
```

## 显形的特殊处理

显形不是给自己点灯，是**拍灭敌方所有隐身灯**：

```
施法者 → 遍历敌方 → 找到所有 is_status_active("stealthed") → 关灯
```

不需要点灯，不需要计时，一次性动作。

## 灯的每帧tick

和buff堆栈一样，每帧倒计时，到时自动关灯：

```gdscript
func _tick_status_lights(delta: float) -> void:
    var to_remove: PackedStringArray = []
    for status in _status_lights:
        _status_lights[status]["remaining"] -= delta
        if _status_lights[status]["remaining"] <= 0.0:
            to_remove.append(status)
    for status in to_remove:
        _status_lights.erase(status)
        print("[Status] %s 状态到期，自动关灯" % status)
```

## 查询接口

```gdscript
# 灯亮没亮？
func is_status_active(status_name: String) -> bool:
    return _status_lights.has(status_name) and _status_lights[status_name].get("on", false)

# 点灯（返回true=成功，false=被免控拦截）
func turn_on_light(status_name: String, duration: float, extra: Dictionary = {}) -> bool

# 关灯（手动，如解控）
func turn_off_light(status_name: String) -> void

# 关掉某种类型的所有灯（如解控关掉所有控制灯）
func turn_off_lights_by_type(light_names: PackedStringArray) -> void
```

## 状态灯 vs Buff堆栈 — 分工对比

| | Buff堆栈（第1步） | 状态灯（第2步） |
|---|---|---|
| 服务标签 | 20个属性标签 | 11个状态/控制标签 |
| 数据结构 | 可叠加的纸条列表 | 不可叠加的开关字典 |
| 计算方式 | 基础值×连乘+求和 | 只有开/关+倒计时 |
| 存储位置 | BuffManager（独立模块） | Player._status_lights |
| 到期行为 | 自动删除纸条 | 自动关灯 |
