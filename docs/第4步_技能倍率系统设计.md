# 第4步：折扣卡（技能倍率系统）设计

## 概念

### 折扣卡 = 技能消耗/CD/效果的临时倍率

三部分：
- **消耗卡**：用技能时能量花费 × 倍率（持续型）
- **CD卡**：技能冷却时间 × 倍率（持续型）
- **效果倍率卡**：下次技能的所有数值标签的 value × 倍率（一次性）

---

## 一、生效范围

### 效果倍率（44/45标签）影响哪些标签？

原则：**只有带数值 value/multiplier 的标签受影响，duration/position/shape 等参数不动。**

#### BALL类（受影响 9个 / 不受影响 8个）

| 受影响 ✅ | params里被乘的值 | 不受影响 ❌ | 原因 |
|-----------|-----------------|------------|------|
| `ball_dmg_up_pct` | value | `ball_tracking` | turn_speed翻倍无意义（行为标签） |
| `ball_dmg_down_pct` | value | `ball_avoid` | 无数值参数 |
| `ball_dmg_up_flat` | value | `ball_boomerang` | return_distance是配置不是强度 |
| `ball_dmg_down_flat` | value | `ball_straight` | 无数值参数 |
| `ball_speed_up_pct` | multiplier | `ball_lockon` | 无数值参数 |
| `ball_speed_down_pct` | multiplier | `ball_spread` | split_count是数量不是强度 |
| `ball_speed_up_flat` | value | `ball_penetrate` | 无数值参数（开关型） |
| `ball_speed_down_flat` | value | | |
| `ball_range_up` | radius, damage_pct | | |
| `ball_range_down` | multiplier | | |

#### FIELD类（全部不受影响 ❌）

场地标签的参数是位置/尺寸/形状/持续时间，翻倍没有意义。比如"障碍物宽度翻倍"不是玩家期望的"效果翻倍"。

| 不受影响 | 原因 |
|---------|------|
| `field_obs_add` | width/height/radius是配置不是强度 |
| `field_obs_clear` | clear_count是数量 |
| `field_terra_change` | friction/bounciness是物理参数 |
| 其余9个 | position/size/duration都不是强度 |

#### PLAYER类（受影响 27个 / 不受影响 14个）

**受影响的（有 value/multiplier 的）：**

| 标签 | 被乘的值 |
|------|---------|
| `player_atk_up/down_pct` / `player_atk_up/down_flat` | value |
| `player_def_up/down_pct` / `player_def_up/down_flat` | value |
| `player_spd_up/down_pct` / `player_spd_up/down_flat` | value |
| `player_res_up/down_pct` / `player_res_up/down_flat` | value |
| `player_vulnerable` | multiplier |
| `player_hp_heal/damage_pct` / `player_hp_heal/damage_flat` | value |
| `player_hp_regen` / `player_hp_dot` | value |
| `player_move_slow` / `player_move_boost` | multiplier |
| `player_energy_gain/cost_pct` / `player_energy_gain/cost_flat` | value |
| `player_energy_max_up/down_pct` / `player_energy_max_up/down_flat` | value |
| `player_spirit_cost_down/up` | multiplier |
| `player_spirit_cd_down/up` | multiplier |

**不受影响的（duration/开关/位置型）：**

| 标签 | 原因 |
|------|------|
| `player_invincible` | duration翻倍=无敌时间翻倍，这不算"效果翻倍" |
| `player_stealth` | 同上，duration型 |
| `player_reveal` | 一次性开关 |
| `player_root` | duration型 |
| `player_unroot` | 一次性动作 |
| `player_stun` | duration型 |
| `player_cc_immune` | duration型 |
| `player_silence` | duration型 |
| `player_disarm` | duration型 |
| `player_spirit_uses_up` | bonus_uses是次数不是强度 |
| `player_spirit_double` | 开关型（就是它自己） |
| `player_spirit_half` | 开关型（就是它自己） |
| `player_teleport` | position型 |
| `player_return` | 一次性动作 |

## 二、实现方法

### 传递链路

```
SpiritSkillTrigger.trigger_skill()
  ↓
  1. 读 player.get_and_consume_next_skill_mult() → 比如 2.0
  2. 把 2.0 存到变量里
  ↓
  _execute_skill_tags()
    ↓
    遍历标签，调 handler.apply_tag_effect(tag_id, params, caster_id)
    ↓
    在调 handler 之前，把倍率注入 params：
    params["_skill_mult"] = 2.0
    ↓
    handler 内部每个标签函数：
    如果这个标签受影响 → params["value"] × params.get("_skill_mult", 1.0)
    如果不受影响 → 原样执行，忽略 _skill_mult
```

### 具体代码位置

**位置1：`spirit_skill_trigger.gd` — 注入倍率**

```gdscript
# _execute_skill_tags 里，构建 params 之后、调 handler 之前
var skill_mult: float = _get_caster_skill_mult(player_id)
tag_params["_skill_mult"] = skill_mult
```

**位置2：`player.gd` — 存储和消费倍率**

```gdscript
var _next_skill_mult: float = 1.0

func set_next_skill_mult(mult: float) -> void:
    _next_skill_mult = mult

func get_and_consume_next_skill_mult() -> float:
    var m: float = _next_skill_mult
    _next_skill_mult = 1.0  # 一次性消费，用完归1
    return m
```

**位置3：`spirit_tag_effect_handler.gd` — 每个受影响标签读倍率**

handler 里已经有了 `_apply_ball_dmg_up(params)` 等函数，加一行读倍率：

```gdscript
func _apply_ball_dmg_up(params: Dictionary) -> void:
    var mult: float = params.get("_skill_mult", 1.0)
    var value: float = float(params.get("value", 0)) * mult
    # ... 后面用 value 代替 params["value"]
```

不受影响的标签（追踪/穿透/无敌/眩晕等）不读 `_skill_mult`，原样执行。

### 消耗/CD折扣卡的实现

消耗和CD折扣不通过 `_skill_mult`，而是通过 Player 上独立的倍率字典：

```gdscript
# use_skill 里
var cost: float = float(skill_data.get("energy_cost", 0)) * get_skill_cost_mult()
# ...
skill_cooldowns[skill_id] = float(skill_data.get("cooldown", 0)) * get_skill_cd_mult()
```
