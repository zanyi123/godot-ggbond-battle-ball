# 第5步：安检门（状态检查）设计

## 概念

### 安检门 = 对外系统的状态检查点

第2步做了**内部检查**（player自己检查自己的灯来限制行动）。
第5步做**外部检查**（别的系统找player时，检查对方的状态灯）。

核心就两个问题：
1. **无敌灯亮的人**，外部系统该怎么对待他？
2. **隐身灯亮的人**，外部系统该怎么对待他？

---

## 现状：已做的 vs 缺的

| 检查点 | 已做（第2/3步） | 缺（第5步补） |
|--------|----------------|--------------|
| 受伤时查无敌 | ✅ take_damage 里查 `is_status_active("invincible")` | — |
| 持续掉血查无敌 | ✅ _process_tick_effects 里查 | — |
| 球选目标时跳过隐身 | ❌ | `_get_nearest_enemy` / lockon / tracking 需跳过隐身者 |
| AI找敌人时跳过隐身 | ❌ | `_find_nearest_visible_enemy` 需跳过隐身者 |
| AOE选目标时跳过隐身 | ❌ | ball.gd AOE范围伤害需跳过隐身者 |
| 球碰撞命中时跳过隐身 | ❌ | `_on_body_entered` + `_check_player_collision_distance` 需跳过隐身者 |
| 球飞过隐身者 | — | 隐身者=不拦截球，球直接穿过 |

---

## 无敌的规则

无敌灯亮时：
- **受伤→伤害变0** ✅ 已实现
- **球仍然会命中他**（球撞到无敌者会停，只是不扣血）
- **AI仍能看到他**（无敌≠隐身）
- **追踪球/锁定仍会瞄准他**（无敌≠不可见）

所以无敌不需要额外的外部检查，第2步的 take_damage 检查已经够了。

---

## 隐身的规则

隐身灯亮时：
- **敌方球飞来→穿过他不拦截**（隐身者不可见，球不会撞上去）
- **追踪球/精准锁定→跳过隐身者**（看不到就锁不上）
- **AI找敌人→跳过隐身者**（看不到就不会追）
- **AOE范围伤害→跳过隐身者**（看不到就不会被波及）
- **队友传球→不受影响**（队友之间不受隐身影响）

---

## 需要加检查的位置（5处）

### 1. `spirit_tag_effect_handler.gd` — `_get_nearest_enemy`

精准锁定和追踪球选目标时，跳过隐身者：

```
现在：遍历敌方 → 不跳过任何人
改成：遍历敌方 → 跳过 is_status_active("stealthed")
```

```gdscript
for e in enemies:
    if e.is_status_active("stealthed"):
        continue  # 隐身者不可见，跳过
    ...
```

### 2. `spirit_tag_effect_handler.gd` — `_get_enemies`

获取所有敌方列表时，过滤隐身者：

```gdscript
for p in players:
    if p and is_instance_valid(p) and p.team == enemy_team and not p.is_defeated:
        if p.is_status_active("stealthed"):
            continue  # 隐身者不可见
        result.append(p)
```

### 3. `ball.gd` — `_on_body_entered`

球碰撞时跳过隐身的敌方：

```gdscript
# 在 "对方球员 → 击中造成伤害" 之前
if player.is_status_active("stealthed"):
    return  # 隐身者球直接穿过
```

### 4. `ball.gd` — `_check_player_collision_distance`

距离碰撞检测也要跳过隐身者：

```gdscript
if p.is_defeated:
    continue
if p.is_status_active("stealthed"):
    continue  # 隐身者不拦截球
```

### 5. `ball.gd` — AOE范围伤害

AOE选范围目标时跳过隐身者：

```gdscript
if p.is_defeated:
    continue
if p.is_status_active("stealthed"):
    continue
```

---

## 不需要改的地方

- `ai_manager.gd` — AI已经有 `_find_nearest_visible_enemy` 函数名暗示了可见性检查，但目前没有查隐身。如果AI后续需要考虑隐身，可以复用同样的 `is_status_active("stealthed")` 检查。**但第5步先不改AI**（AI系统较复杂，隐身对AI的影响需要单独测试）
- 无敌 — 不需要额外外部检查，take_damage里已经挡了

---

## 显形的处理

显形标签 `player_reveal` 不是给自己点灯，是拍灭敌方所有隐身灯。
这个属于第6步填标签函数时实现，第5步只建好检查点。

## 测试用例清单

| # | 测试名 | 验证什么 |
|---|--------|---------|
| 1 | 隐身者被get_enemies跳过 | _get_enemies返回列表不含隐身者 |
| 2 | 隐身者被get_nearest_enemy跳过 | _get_nearest_enemy返回null（全隐身时） |
| 3 | 隐身者不影响队友 | get_enemies不含己方，但己方能看到隐身的队友（逻辑本身team过滤已保证） |
| 4 | 显形后可被选中 | turn_off_light("stealthed")后能被get_enemies找到 |
| 5 | 无敌者仍可被选中 | is_status_active("invincible")不影响get_enemies结果 |
| 6 | 多人部分隐身 | 3个敌人中1个隐身 → get_enemies返回2个 |

## 待确认

1. **隐身者是否被队友的AOE伤害波及？**
   - 建议：**是**。隐身只对敌方生效，AOE是己方技能时不应该因为敌方隐身就排除。但当前AOE是"以被击中敌方为中心对其他敌方造成伤害"，所以AOE目标全是敌方，隐身者应被跳过。
   - 如果以后有"对己方AOE"的标签，那时再判断。

2. **追踪球的目标隐身后，球怎么处理？**
   - 建议：**丢失目标，改为直飞**（tracking_target 设为 null）
   - 球已经在飞了，目标突然隐身→球失去追踪，直线飞完剩余距离
